# Soup Simulator Implementation Plan (v2 — post plan-review)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Measure delivery-ratio and airtime-cost vs node density for the polleneus pure-flooding "uniform soup", validated against percolation ground truth so the located cliff is real, not an artifact.

**Architecture:** Deterministic, time-stepped discrete simulation. Nodes move (static-uniform primary, RWP overlay); contacts are timed **analytically** (closest-approach per leg, dt-independent); blobs flood **multi-hop** under a density-aware airtime budget, bounded buffer, and sliding-window seen-record. A metrics layer holds the only knowledge of sender/recipient. A scenario runner sweeps density with **per-replication** statistics and validates against an **independent union-find component oracle** + the **susceptibility-peak** threshold locator.

**Tech Stack:** Python 3.11+, numpy. matplotlib optional. pytest.

## Global Constraints
- Python 3.11+; deps = numpy (core), matplotlib (optional, import-guarded); pytest.
- One injected `numpy.random.Generator`; **no module-global RNG** (lint). Substreams via `SeedSequence.spawn`.
- **Invariant guard:** `model`/`engine`/`policies` read only `{id, created_at, ttl, size}`; `sender`/`recipient` live only in metrics (lint).
- Absolute TTL; eviction = oldest-by-creation cohort + randomized tie-break (never closest-to-TTL).
- **Edge convention (pinned):** an edge exists iff center-to-center `dist ≤ r`; expected degree `d = λπr²` — this is the constant the continuum-percolation threshold **d_c ≈ 4.51** is defined against.
- **Every reported number is an UPPER BOUND** on real-world delivery (README must say so).
- Code under `sim/`; package `soup_sim/`; tests `sim/tests/`.

### Task 12: Percolation-validation GATE (union-find oracle + susceptibility peak)
**Files:** `tests/test_integration_percolation.py`; (helper) `soup_sim/percolation.py`
**Interfaces:** `largest_component_fraction(positions,r,w,h,boundary)->float` and `susceptibility(positions,r,w,h,boundary)->float = sum_{c != largest} |c|^2 / N` via union-find over `neighbor_pairs`; `same_component_pairs(...)->set[(i,j)]`.
- [ ] **Step 1: Write the gate tests (the keystone):**
  - **Independent-oracle KAT:** in static + B=∞ + TTL=∞ + buffer=∞, the set of delivered (src,dst) pairs from `Engine.settle_static_fixpoint()` **exactly equals** `same_component_pairs` from union-find. (Cross-checks the whole stack against an independent reachability computation; catches any exchange bug.)
  - **Threshold locator (the real 4.51 check):** sweep mean-degree over a Poisson ensemble (`N ~ rng.poisson(λWH)` per rep) on a torus; assert the **susceptibility χ(d) peaks** in `[4.0, 5.2]` (finite-size threshold locator), AND the giant-component knee satisfies `S(4.0) < 0.1` and `S(5.5) > 0.3`. **Do NOT** assert delivery=0.5 ≈ 4.51 (that crossing is ~d 6–7).
  - **Aggregation (pin):** χ(d) and S(d) are the **ensemble MEAN over reps** of the per-rep intensive quantities (χ=Σ|c|²/N over non-largest components, S=|largest|/N) before argmax / knee bounds; never pool raw |c|² across reps with differing N.
  - **Min system size (pin):** target N on the order of a few thousand (≈ the n=5000 torus degree test) so finite-size fluctuations don't violate S(4.0)<0.1 or broaden the susceptibility peak outside [4.0,5.2].
  - **Regime invariance:** the static gate delivery result is invariant to `measure_window` and `dt` (it is a component property via fixpoint).
