# polleneus v0.3 → v0.4 — Token-Source Design + Red-Team (digest)

**Date:** 2026-06-25
**Method:** 4-stage pipeline (12 agents). **Design** (6 experts each invent one token source) → **Select** (lead cryptographer scores all six, synthesizes the winner) → **Red-team** (adversaries attack all four v0.3 primitives with the chosen source baked in) → **Synthesize** (verdicts + v0.4 changes).
**Outcome:** all "now" changes folded into spec **v0.4**.

> Full raw findings (all candidates + per-primitive attacks/fixes) are archived in **`2026-06-25-redteam-v0.3-raw.json`**.

## Part 1 — The token source (the open problem from §17)

**Brief:** where do the link-local relay tokens *come from*, given they must be Sybil-resistant (R1), farm-resistant (R2), anonymity-preserving (R3), serverless/offline (R4), no-global-state (R5), BLE-buildable (R6), and incentive-compatible (R7)?

### Candidate ranking
| Candidate | Score | One-line |
|---|---|---|
| **Witnessed Relay Reciprocity (WRR)** | 8.6 | Earn tokens by relaying novel traffic to diverse strangers; only *preserves-all*, best R7, lowest build cost; weak R1 floor + sparse-crowd liveness gap. |
| **Hourglass (PoSW + ZK nullifier)** | 8.0 | Self-minted sequential Proof-of-Sequential-Work spent as a ZK nullifier; *eliminates* the farm surface, best acquisition anonymity + per-device physics cap; worst R7, heavy ZK-wrapper risk. |
| Proof-of-Encounter (KVAC co-sign) | 6.8 | Earn by meeting diverse people; strong R3/R7; leans on weak BLE distance-bounding; mobile-farm residual. |
| Quorum-Witnessed (Coconut threshold) | 6.0 | k-of-n blind-signed by nearby peers; production-grade core; sparse-crowd liveness failure; heuristic PHY anti-farm. |
| Witnessed Sweat (co-presence × work) | 5.5 | Product-of-two-scarcities; brittle shared-MAC-key security argument. |
| VOUCH (social-graph vouched) | 4.2 | Sybil-bound by social degree; **critical** shared-key SPOF; bends several invariants. |

### Winner — "Reciprocity Hourglass" (a deliberate blend of the top two)
**WRR spine × Hourglass primitive.** One token format, two ways to earn it:
- **Reciprocity path (cheap, prosocial):** relay *novel* blobs to *diverse* strangers → credit → mint at steeply reduced sequential-PoSW difficulty. Price → 0 near the percolation cliff (relaying rewarded when relays are scarce — attacks the tragedy-of-the-commons).
- **Sweat path (always available):** self-mint by paying full background sequential PoSW (emits nothing on the wire). Keeps the system live in sparse crowds.
- Spent as a **Fiat-Shamir ZK nullifier in the BLE handshake**, never in the blob.
- **Safety rule that fuses them without reopening either's loophole:** every token still costs sequential wall-clock (reduced difficulty floored > 0; discount strictly below the token's re-priced slot value).
- Chose **hash-based PoSW over a class-group VDF** (phones have hash hardware; no trusted setup, no group of unknown order).

## Part 2 — Red-team of the four v0.3 primitives (with that source baked in)

| Primitive | Verdict | Top attack | Top fix (now in v0.4) |
|---|---|---|---|
| **A — Link-local relay tokens** | ⚠️ **dented** | **Per-neighbor nullifier → one token = ~D slots** (fresh valid nullifier per distinct neighbor; double-spend memory is only local per-neighbor-per-session). Collapses the rate-limit. | **Token-anchored nullifier** `nf=H("nf"‖s)` + **epidemic-gossiped seen-set** → "≈1 slot venue-wide." Plus token TTL + re-priced discount. |
| **B — Crypto-shred-on-read** | ⚠️ **dented** | "Never decryptable again" is **false** — Bloom-Filter Encryption has false-positive *retention* and hundreds-of-MB keys; doesn't fit a weak phone at venue scale. | **Time-ratcheted FS** (keys deleted by the clock, not by reading — also kills the seizure/message-suppression hole); guarantee **probabilistic (≤ p)**, surfaced in UX; SE/TEE-wrapped keys. |
| **C — VDF age-floor** | ❌ **broken** | The age-floor is a compute **lower** bound, not a wall-clock **upper** bound → can't expire anything; backdating still works; and it contradicts the chosen hash-PoSW. | **Drop the standalone VDF**; unify mint-cost on hash-PoSW; expiry is the signed TTL only; precompute defeated by the per-session nullifier anchor. |
| **D — Origination gate** | ⚠️ **dented** | The hard gate **self-deadlocks** in sparse/blackout venues (percolation cliff biting anonymity); "send anyway with a warning" is itself a tell; Poisson buys little once device-linkage ≈ 1.0. | **Probabilistic, time-bounded origination license** (floored > 0, never deadlocks) + **manufacture local cover** in sparse mode; report anonymity as a **measured source-estimator probability**, not K; fixed venue-wide Poisson rate. |

**Token-source verdict:** **dented, not broken** — architecture is right; survives only *with* the four §9 fixes (chiefly the token-anchored nullifier). Without them, primitive A provides almost no rate-limit.

## Biggest residual
Funded-adversary **device count** — structurally unfixable without bending an invariant. (1) **Cloud/botnet sweat-minting** (sequentiality caps per-device, not core count; no global difficulty oracle to raise N adaptively). (2) **Spread-out farm at crowd density** (K real radios serving real strangers legitimately clears the diversity gate — the honorable O(crowd) bound). The token-anchored nullifier keeps this a dent by forcing a farm to pay sequential wall-clock per slot AND genuinely improve the mesh to mint cheaply. Honest posture: flooding costs **O(rented cores + K radios), not impossible** — a cost bound, not a wall, labeled in-app.

## Soul-check
Holds. Recommended fixes preserve all seven invariants. The win is honesty applied to two overclaims the *app* would otherwise have made: "never decryptable again" → probabilistic + clock-ratchet, and "VDF age-floor backstops immortality" → deleted (only the TTL expires anything). The two standing truths got *sharper*: the percolation cliff now visibly bites the anonymity layer, and device-linkage ≈ 1.0 forces a *measured* anonymity probability instead of a flattering ratio.

## Open questions promoted to active research (§17)
1. Exact discount curve (diverse-novelty → reduced PoSW) provably keeping cost > r×slot-value across density, no global oracle.
2. ZK-spend wrapper verify cost on low-end Android (Poseidon vs Rescue vs Bulletproofs) vs BLE MTU/battery.
3. N_sweat sizing so cloud-farm minting is uneconomic at 50k-phone scale without honest battery harm.
4. Byte-indistinguishability of sweat- vs reciprocity-minted tokens at spend.
